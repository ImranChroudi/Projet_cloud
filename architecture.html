Détail des services

🖥️

Frontend

:5173

Interface utilisateur construite avec React 19 et Vite. Elle offre un tableau de bord, la gestion de projets avec CRUD complet, un Kanban board drag-and-drop pour les tâches, la gestion des catégories et des utilisateurs, et un chat temps réel via Socket.IO. L'authentification se fait via JWT stocké dans localStorage.

React 19 Vite React Router v7 Axios Socket.IO Client react-beautiful-dnd react-toastify Lucide Icons

auth-service

:5001

Service d'authentification basé sur MySQL. Gère l'inscription et la connexion des utilisateurs avec hashage bcrypt des mots de passe. Génère des tokens JWT (expiration 1h). Fournit aussi un CRUD utilisateur (bloquer, rôles).

service-messages (+ notifications)

:3003

Microservice de messagerie temps réel et de notifications. Combine une API REST pour charger l'historique des messages et Socket.IO pour l'envoi/réception en temps réel. Chaque projet est un salon (room). Intègre le système de notifications : les autres services (gestion-projects, service_tache) envoient des notifications via POST, et le frontend les récupère et les marque comme lues via ce même service. L'authentification se fait par JWT dans les headers REST et dans le handshake Socket.IO.

Express MongoDB Mongoose Socket.IO JWT

Endpoints REST — Messages

GET /messages/:projectId
POST /messages

Endpoints REST — Notifications

GET /notifications — Récupérer les notifications
POST /notifications — Créer une notification (appelé par les autres services)
PUT /notifications/:id/read — Marquer une notification comme lue
PUT /notifications/read-all — Marquer toutes comme lues

Événements Socket.IO

EMIT joinProject(projectId)
EMIT leaveProject(projectId)
EMIT sendMessage({ projectId, text })
ON newMessage(message)
ON notification(notif) — Notification temps réel

Déclencheurs de notifications

TRIG Message envoyé → notification automatique aux membres du projet
TRIG Création d'un projet → notifie via POST depuis gestion-projects
TRIG Création d'une tâche → notifie via POST depuis service_tache

🔗 Communication inter-services

Frontend :5173 → auth-service :5001 HTTP (Axios) Login, Register
Frontend :5173 → gestion-projects :3000 HTTP (Axios) Projets, Catégories, Users, Tasks
Frontend :5173 → service_tache :5003 HTTP (Axios) Kanban CRUD tâches
Frontend :5173 → service-messages :3003 HTTP + WebSocket Chat temps réel + historique + Notifications
gestion-projects :3000 → service-messages :3003 HTTP POST /notifications Création d'un projet
service_tache :5003 → service-messages :3003 HTTP POST /notifications Création / assignation tâche

🔗 Relations entre modèles

Message.user → User._id Chaque message est lié à l'utilisateur qui l'a envoyé (populate: username, email)
Message.projectId → Project._id Chaque message appartient à un projet (salon de chat)
Task.assignedTo → User._id Chaque tâche est assignée à un utilisateur
Task.projectId → Project._id Chaque tâche appartient à un projet
Task.comments[].user → User._id Chaque commentaire est lié à son auteur
Project.createdBy → User._id Chaque projet est créé par un utilisateur
Project.idCategory → Category._id Chaque projet appartient à une catégorie
Notification.userId → User._id Chaque notification est destinée à un utilisateur
Notification.projectId → Project._id Chaque notification est liée à un projet
